- **清单驱动架构**：采用双层 Manifest 设计，根目录 `.cursor-plugin/marketplace.json` 定义市场元数据与插件注册表，各插件子目录 `.cursor-plugin/plugin.json` 声明具体能力（MCP/Skills/Rules）。
- **组件化能力封装**：以 `plugins/<plugin-id>` 为独立单元，内部按官方约定组织 `mcp.json`（外部工具桥接）、`skills/`（Agent 工作流指引）及 `rules/`（持久化约束）。
- **自动化校验闭环**：通过 `scripts/validate-template.mjs` 实现本地与 CI（GitHub Actions）双重校验，强制检查 JSON 合法性、路径安全性、Frontmatter 完整性及名称一致性，确保市场配置的健壮性。
- **安全隔离机制**：敏感配置（如 API Token）在仓库中仅保留占位符，依赖成员本地环境或密钥管理系统注入，防止机密信息泄露。
