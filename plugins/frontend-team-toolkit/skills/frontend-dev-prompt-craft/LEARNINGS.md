## 2026-06-09 [PAGE+UI] 异地推荐字段条件渲染提示词生成

**场景**: 用户需要为人员管理模块的「人员关系」区域写开发提示词，涉及后端新增互斥字段（`refereeId` vs `remoteRefereeId`），前端根据字段有无值动态切换 label 和数据绑定，并在异地推荐场景下清空育成人/渠道经理展示。

**反馈**: 效果好。追问流程顺畅（路由→接口字段截图→组件库），生成的提示词可直接用于 `openspec-contract-authoring` 生成契约四文件。提示词中包含了具体文件路径、行号范围、字段矩阵和验收标准，AI 编码工具可直接理解。

**改进点**:
1. 现有 PAGE 模板（P1-P4）主要面向「新增页面/新增功能」，缺少「现有字段条件映射」场景的模板。本次实际更接近一种新的任务子类型：`FIELD-MAP`（字段条件渲染/映射），建议新增模板覆盖。
2. 当用户提供接口截图时，技能应主动解析截图中的字段信息并纳入提示词，而不是等待用户手动输入。
3. 与 `openspec-contract-authoring` 技能的衔接很顺，建议在 SKILL.md 中显式提到「生成的提示词可作为 openspec 四文件的输入」。

**是否同步到 SKILL.md**: 是（待积累 3 条同类反馈后新增 FIELD-MAP 模板）
