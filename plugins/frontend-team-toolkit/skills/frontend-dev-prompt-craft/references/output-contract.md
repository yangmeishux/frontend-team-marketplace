# Output Contract — 前端开发提示词工程

## 输出格式

每次执行必须输出以下内容：

### 1. 任务类型识别结果

```
📋 任务类型：{{type_code}} — {{type_name}}
📎 组合类型：{{combo_types}}（如有多种）
```

### 2. 信息完整度检查

列出所有必填项的收集状态：

| 必填项 | 状态 | 值 |
|--------|------|-----|
| 项目/文件路径 | ✅ / ❌ 缺失 | 具体值 |
| 功能描述 | ✅ / ❌ 缺失 | 具体值 |
| 数据来源 | ✅ / ❌ 缺失 | 具体值 |

如有缺失项，**必须追问**，不得编造。

### 3. 生成的结构化提示词

```markdown
## 任务：{{task_title}}

**类型**：{{type_code}}
**涉及文件**：{{file_list}}
**接口**：{{api_list}}

### 需求描述

{{structured_requirement}}

### 技术约束

{{constraints}}

### 验收标准

- [ ] {{acceptance_1}}
- [ ] {{acceptance_2}}
- [ ] {{acceptance_3}}
```

### 4. 使用说明

简要说明生成的提示词如何使用，是否需要拆分步骤。

## 质量红线（must）

- 生成的提示词必须包含具体的文件路径或路由路径（不得用 TODO 占位）
- 生成的提示词必须包含至少一条验收标准
- 如涉及接口，必须包含接口地址和关键参数说明
- 提示词必须可直接复制粘贴给 AI 编码工具使用
- 信息不足时必须追问，不得编造路径或接口地址

## 禁止行为（must_not）

- 不得生成空泛的「请帮我开发一个页面」类提示词
- 不得在信息不足时直接生成完整提示词（应追问）
- 不得遗漏用户提供的关键上下文（如 UI 稿链接、接口文档链接）
- 不得将多个独立任务合并为一条巨型提示词（单一职责）
- 不得修改用户提供的接口地址、文件路径等事实信息
