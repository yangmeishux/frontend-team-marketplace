# Vue 2 → Vue 3 组件迁移（依赖驱动 · 行为等价）

## 执行协议（两阶段 · 方案 B）

**默认顺序**（除非用户明确说「跳过规划、直接改码」并自担遗漏风险）：

1. **第一轮交付物**（可不填业务细节，但**表头与骨架须齐**）：**`reference.md` §8 一页纸模板**中的元信息、依赖登记册、因果链表、隐式耦合表；并对 **§10 易漏项扫雷** 逐项勾选或标注「待确认 / 不适用」。  
2. **第二轮起**再批量修改业务源码；若第一轮发现闭包未收敛，应先补登记再扩写代码。

首轮允许只做 **检索与列表**（路径、import 边），不强制已写出完整 Vue 3 实现。

---

## 工程就绪（迁移前 · 方案 C）

在开始「阶段一」扩张闭包之前，在**目标 Vue 3 宿主工程或隔离包预期宿主**完成下列核对（**展开说明与条目详见 `reference.md` §9**）：

- [ ] **`package.json`**：Vue、UI 库、状态库、请求库、TS 等版本与团队基线一致或可说明差异。  
- [ ] **锁文件 / 包管理器**：与安装与 CI 一致（pnpm / npm / yarn）。  
- [ ] **构建入口**：存在 `vite.config.*` / `webpack` / `rsbuild` 等之一，并知悉**开发/生产构建命令**。  
- [ ] **TS 与路径**：若启用 TS，知悉 `tsconfig` 中 **paths** 与 strict 相关约束是否与迁移策略冲突。  
- [ ] **验证手段**：至少明确一项可重复命令（如 `pnpm build`、`pnpm test`、单测）用于迁移后 **结构门** 佐证。

核对结论可简述记入一页纸「元信息」或 PR 描述。

## 技能边界（单一职责）

只做：**理清依赖与因果 → 将指定 Vue 2 页面/组件迁入 Vue 3，并保持对外可见行为一致**（可审计流程）。

不包含：与迁移无关的新功能设计；全应用级路由/权限/构建重写（仅处理与目标直接相关的触点）；强行整体换状态库范式（除非用户单独立项）。

**长示例、对照表、完整扫雷/校验/坑点**：**`reference.md`**（§1–§7 语法与附录，§8 模板，§9–§12 延展）。**增量 Git 基线**：**`MIGRATION_BASELINE.md`**（可选）。

## 迁移原则（摘要）

1. **闭包完整**：闭包外引用须纳入、改宿主约定或 README，禁止悬空 import。  
2. **行为等价优先**：先有输入/输出与副作用说明，再动代码；非等价须有记录与评审。  
3. **路径可归因**：隔离包落地相对路径或文档化别名；就地升级则全仓引用一致。  
4. **范式统一**：默认 **`<script setup>` + 组合式 API**（除非项目另有规范且用户指定）。  
5. **依赖边界清晰**：`package.json` 与真实 import 一致；无密钥进库。  
6. **可验证**：通过 **双重闸门** 与 **核心校验清单**（完整版见 **reference §11**）。

---

## 防遗漏（精简）：三问 · 闸门 · 登记

**闭包三问**（未收敛前不宜大规模改写）：① 沿 import+模板能否走到叶节点且无未登记全局依赖？② 模板每个组件/指令在 Vue 3 侧有出处？③ 每个非 props 副作用是否挂在可追踪链上？

**双重闸门**：

| 闸门 | 通过的判据 |
|------|------------|
| **结构门** | 闭包内引用可解析；模板与脚本组件来源一致；`package.json` 与 import 一致 |
| **行为门** | **因果链登记表**（§8）**逐行**对照 Vue 2，或有已批准差异说明 |

**依赖登记册、因果链、闸门勾选项**：填 **`reference.md` §8**；**模板边补扫**（`$t`、`@import`、`dynamic import` 等）见 §8.2 说明。

**易漏细节清单（完整勾选表）**：**`reference.md` §10**。

## 迁移流程（七阶段摘要）

各阶段细节、语法表、子组件/form 注意事项见 **`reference.md` §1–§6** 与本节呼应。

1. **分析与拆解**：入口 SFC **BFS/DFS** 扩张闭包；**每文件单独扫 `<template>`** 补边；隐式耦合记入登记册；落 **因果链登记表**；拓扑上先底座再创页面。  
2. **目录落位**：就地遵循工程规范；隔离包参见 **reference** 与下文「依赖清单」。  
3. **语法与运行时**：Options → 组合式 API、**`v-model`/emit**、`:deep()` 等见 **reference §5**；大版本 UI 差异单列对照表。  
4. **子组件与 ref**：`defineExpose`、`unref`、watch 对齐 Vue 2 语义；见 **reference §2**。  
5. **跨文件子树**：无环；store 用 **`inject`** 可降级模式见 **reference §3**；私包写入 `package.json`。  
6. **宿主集成**：去掉仅为注入而存在的 props；同步父级事件/插槽/`v-model`；README + `package.json` 与登记册勾连。  
7. **完备性复核**：白名单递归、模板反向打勾、因果链逐行最小复现推演、删码须有因果作废说明；通过后视为闸门满足。

## 依赖清单

按闭包真实 `import` 生成 `package.json`；隔离包至少：

```json
{
  "peerDependencies": {
    "vue": "^3.3.0"
  }
}
```

其余逐条加入；**禁止**提交密钥。细则见 **reference §7**。
