# 迁移 Git 基线（增量同步）

在同一 Vue 2 源码目录会持续迭代时，用 **Git 提交基线** 限定「仅需再次迁移的范围」，减少重复改写。

## 使用步骤

1. **首次迁移完成并合入**：记录此时源目录对应的 **commit SHA**（或 tag）与源路径。
2. **后续同步**：列出基线以来的变更文件，仅对这些文件及其新产生的依赖边做 Vue 3 侧更新。

## 基线信息（由项目填写）

| 项目 | 值 |
|------|-----|
| 基线 commit |  |
| 基线标注日期 |  |
| Vue 2 源目录（相对仓库根） |  |
| Vue 3 目标目录或分支说明 |  |

## 列出变更文件（示例命令）

```bash
# 将 <BASE_SHA>、<SOURCE_PATH> 替换为实际上下文
git log <BASE_SHA>..HEAD -- <SOURCE_PATH>

git log <BASE_SHA>..HEAD --name-only --pretty=format: -- <SOURCE_PATH> | sort -u
```

将命令输出作为「待审查迁移」集合；若在 Vue 3 侧有手工调整，需在 PR 或内部文档中注明，避免下次增量覆盖时丢失。
