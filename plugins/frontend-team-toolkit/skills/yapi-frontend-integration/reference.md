# YApi 前端对接 · 参考手册

与 **`SKILL.md`** 配套：字段说明、映射、类型与排障。**建议顺序**：先读 `SKILL.md`（两阶段协议与边界）→ 本文 §2–§5 → 遇问题时 §7。

---

## 1. MCP 工具名与版本

插件依赖 **`yapi-devloper-mcp`**（见 `mcp.json`）。各 Cursor 版本/包版本下 **Tool 名称与参数** 以 **MCP 面板中展示的描述** 为准。下文以 `get_api_list`、`get_api_desc` 为**惯用名**；若实际名称不同，等价替换即可。

若包还提供 **`create_api` / `update_api`**：仅在用户明确要求、且确认有 YApi 写权限时使用，避免误改线上文档；默认技能路径为 **只读拉文档 + 写前端代码**。

## 2. 获取接口文档

### 2.1 输入形态与动作

| 用户输入 | 建议动作 |
|----------|----------|
| **分类 / 目录 URL**（含 `cat_` 等） | 调 **列表类** 工具拉该分类下接口；条数多时请用户挑选或补充关键词 |
| **数值 `apiId`** | 调 **详情类** 工具直接取接口定义 |
| **接口中文/英文名称** | 先在有范围的列表结果中筛选；若无唯一匹配，**不要猜测**，列出候选项让用户选 |

### 2.2 `get_api_desc` 关键字段（惯用名）

不同导出可能略有差异，以返回 JSON 为准，常见包括：

- **method**、**path**：HTTP 方法与路径（注意是否含全局 `baseURL`）  
- **req_query**：Query 参数  
- **req_headers**：Header 要求  
- **req_body_other**：请求体（常为 JSON Schema 或示例）  
- **res_body**：响应体（常为 JSON Schema 或示例）

Path 中可能含 **`:id`** 等占位符，封装时做成函数参数。

## 3. YApi 字段 → 前端请求（axios 惯例）

| 文档侧 | axios / 封装层 |
|--------|----------------|
| GET + `req_query` | `params: { ... }` |
| POST/PUT/PATCH + JSON body | `data: { ... }`，`Content-Type` 按项目实例默认 |
| `req_headers` | 在拦截器或单次 `headers` 传入（勿提交密钥到 Git） |
| 路径参数 `/user/:id` | 模板替换后拼到 `url` |
| 文件上传（若文档为多表单） | 使用 `FormData`，勿混用 JSON `data` |

---

## 4. TypeScript 与 JSON Schema（可选但推荐）

- 若 `res_body` / `req_body_other` 提供 **JSON Schema**，可推导 **interface / type**，字段名以 Schema 为准。  
- 若仅 **示例 JSON**，可据示例生成类型，并在注释中注明 **「推断自 YApi 示例，以联调为准」**。  
- **不要**为文档未出现的字段补全「想象中」的可选属性，除非业务确认。  
- 团队若有 **openapi-typescript**、**yapi-to-ts** 等脚本，优先遵循项目既有命令，本技能不强制工具链。

---

## 5. 实现 API 封装与页面调用

1. **定位项目请求基座**：现有 `src/api/request.ts`、`axios` 实例、统一错误处理与 **业务 code** 解析方式。  
2. **单接口单方法或按模块文件**：命名语义化（动词+资源，或与后端 operationId 对齐）。  
3. **错误与 loading**：遵循项目约定（如 `ElMessage`、`toast`、全局错误边界）。  
4. **页面侧**：按 `res_body` 结构解构；列表注意分页字段名与文档一致。

---

## 6. 写回 YApi（非默认）

仅当用户明确要求「在 YApi 上建/改接口」且 MCP 暴露写能力时：

- 先确认 **环境/项目** 正确，避免写到生产项目  
- **小步变更**，变更后再次 `get_api_desc` 核对  
- 在对话或 PR 中记录 **改了哪些接口 id**

---

## 7. 故障排查

| 现象 | 检查 |
|------|------|
| 工具报认证失败 | `YAPI_BASE_URL`、账号密码、该账号在项目中的权限 |
| 始终拉不到列表 | URL 是否完整、分类 id 是否正确、网络与内网 VPN |
| 返回字段与线上不一致 | 以联调为准；推动后端更新 YApi 或前端临时兼容层（需评审） |
| Cursor 内看不到工具 | MCP Server 是否启用、`npx` 与 Node 是否可用 |

## 8. 快速参考

| 用户说法 | Agent 优先动作 |
|----------|----------------|
| 「对接 xxx 接口」 | 能解析 id/链接则拉详情；否则列表工具 + 用户选择 |
| 粘贴 YApi 分类链接 | 列表 → 选接口 → 详情 → 两阶段交付后落码 |
| `apiId=12345` | 详情 → 摘要 → 确认 → 封装 |

---

## 9. 安全与隐私

- **勿**把 YApi 密码、个人 Token 写入仓库或贴进公开 PR。  
- 示例代码、日志、截图中对 **手机号、证件、银行卡** 等打码。  
- 若接口响应含敏感字段，前端仅展示业务所需列，并遵守团队数据分级要求。
